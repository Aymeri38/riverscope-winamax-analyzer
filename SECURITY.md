# Sécurité

## Versions prises en charge

Seule la branche `main` est maintenue. Ce projet est un outil post-session : toute régression permettant une analyse pendant que `Winamax.exe` fonctionne est considérée comme critique.

## Signaler une vulnérabilité

Utilisez de préférence **Security → Report a vulnerability** sur GitHub afin de créer un signalement privé. Si cette option n’est pas disponible, ouvrez une issue minimale demandant un canal privé, sans publier le détail exploitable.

Ne joignez jamais à un signalement : historique Winamax réel, pseudo, identifiant de tournoi ou de main, chemin local, base SQLite, sauvegarde, invitation, jeton, fichier DPAPI, clé TLS, clé API ou référence privée de l’accord déclaré par l’hôte.

Indiquez la version ou le commit, le système d’exploitation, l’impact observé et une reproduction basée uniquement sur des données synthétiques. Ne testez pas une vulnérabilité sur le hub ou les données d’une autre personne sans autorisation explicite.
